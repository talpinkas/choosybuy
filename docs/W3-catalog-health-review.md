# W3 — סקירת פערי סיווג והצעות תיקון

נכתב 2026-06-21. מבוסס על שחזור הבאג בפועל דרך נתיב `get-pool` האמיתי + שלושה אודיטים מקבילים (Terminal X, NEXT, Shilav+Fox).

הסיווג שעליו נשען הסינון הוא שלושה שדות לכל מוצר: **צבע, מגדר, קטגוריה**. כל פער למטה מסומן: חומרה, מה נמצא (עם דוגמה), שורש הבעיה, ההצעה שלי, ומאמץ/סיכון.

---

## באג 1 — צבע: מילת הדפס/הדגשה גוברת על צבע הבסיס

**חומרה: גבוהה.** זה הבאג שדיווחת עליו (חולצה אדומה 2-8 שמחזירה צבע אחר).

**מה נמצא.** מוצר NEXT עם התווית

```
בז'/חום בעיטור דובדבן
```

חוזר תחת מסנן **אדום**, כי ה-regex של אדום מכיל `דובדבן` (cherry) ורץ לפני חום. אותו דפוס מסווג תוויות עם לוכסן כמו

```
ורוד/כחול
```

תחת כחול במקום הצבע המוביל ורוד.

**שורש הבעיה.** `colorFamily` ב-

```
api/get-pool.js
```

עובר על משפחות הצבע בסדר עדיפות קבוע ומחזיר את הראשונה שמתאימה. לכן מילת קישוט (`דובדבן`, `פרחוני`) או צבע משני אחרי לוכסן יכולים לנצח את צבע הבסיס הדומיננטי. ההערה בקוד אומרת שצבע בסיס אמור לנצח הדפס — אבל המימוש לא עושה את זה.

**הצעת תיקון.** להחליף את סדר-העדיפות בכלל **"הצבע המוביל מנצח"**: בוחרים את משפחת הצבע שמופיעה הכי שמאל בתווית (התווית כמעט תמיד פותחת בצבע הדומיננטי), והדפס/`מצויר` רק כברירת מחדל אם אין אף צבע ממשי. חריג חשוב: מילות חומר כמו

```
ג'ינס · denim · מלאנ'
```

לא נחשבות "צבע מוביל" (אחרת `ג'ינס שחור` היה הופך לכחול במקום שחור).

**השפעה.** ~275 מוצרים (3.1%) משנים סיווג, הרוב המכריע לטובה.

**מאמץ/סיכון.** קובץ אחד (`get-pool.js`), בלי rebuild. סיכון נמוך. צריך בדיקת רגרסיה קצרה על מקרי החומר (`ג'ינס`, `מלאנ'`).

## באג 2 — מגדר: חצאיות Shilav מוצגות לבנים

**חומרה: גבוהה** (הסוג הכי גרוע — פריט בנות מול בן).

**מה נמצא.** ~9 חצאיות וחליפות-חצאית ב-Shilav מופיעות תחת **בן** (`shilav-boy-2-8-bottoms`, `shilav-boy-0-2-bodysuits`).

**שורש הבעיה.**

```
tools/build-shilav.js
```

מסווג `חצאית` כקטגוריית bottoms, ול-Shilav אין מקור מגדר — אז הפונקציה `gendersOf()` נופלת ל-unisex ומציגה לשני המגדרים. כפיית `girl` קורית רק כשהקטגוריה היא `dresses`, וחצאית אינה dress.

**הצעת תיקון.** ב-`gendersOf()`, לפני נפילת ה-unisex: אם הכותרת מכילה

```
חצאית
```

להחזיר `['girl']`. אחריו rebuild של Shilav (Shopify פתוח, ~10 שניות).

**מאמץ/סיכון.** שורה אחת + rebuild אוטומטי. סיכון נמוך.

---

## באג 3 — קטגוריה: מוצרים בודדים תחת קטגוריה שגויה ב-TX ו-NEXT

**חומרה: בינונית-נמוכה** (~15 מוצרים בסך הכול).

**מה נמצא.**
- Terminal X מתייק ~8 אוברולים (`אוברול`) ו-~5 חולצות תחת bottoms — כי הטקסונומיה של TX מכניסה אוברולים ל-`pants-jumpsuits`.
- שני מכנסי כדורעף של Nike ב-NEXT יושבים בקטלוג swim (טעות נתונים מההפקה הידנית).

**שורש הבעיה.** הקטגוריה ב-TX נקבעת לפי מזהה-הקטגוריה של החנות, לא לפי הכותרת — אז כשהחנות מתייקת לא נכון, גם אנחנו. ב-NEXT זו טעות חד-פעמית בסנאפשוט הידני.

**הצעת תיקון.** רשימת override קצרה לפי כותרת בזמן build: אם כותרת ב-bottoms מכילה `אוברול`/`סרבל` → bodysuits, ואם מכילה רק `חולצ` → tops. את שני פריטי ה-Nike להסיר בהפקה הבאה של NEXT (אין מה לתקן בקוד).

**מאמץ/סיכון.** נמוך, אבל ההחזר קטן. אפשר לדחות בלי לפגוע בפיילוט.

## ניקוי כלי האודיט (לא באג נתונים — רעש בדוח)

**חומרה: נמוכה, אבל משפר תחזוקה.**

**מה נמצא.** המספרים המפחידים באודיט הם false positives:
- TX "dresses←bottoms" (76) הן חצאיות — נכון שיושבות ב-`dresses-skirts`.
- NEXT "swim←bottoms/tops" (~304) הם מכנסי שחייה וחולצות גלישה — בגדי ים לכל דבר.

**שורש הבעיה.** בדיקת הקטגוריה ב-

```
tools/audit-catalogs.js
```

היא היוריסטיקת מילות-מפתח גסה. `KW.swim` מזהה רק `בגד ים`, אז כל פריט שחייה אחר (`מכנסי שחייה`, `חולצת גלישה`) נדלק כאי-התאמה.

**הצעת תיקון.** להרחיב את `KW.swim` ל-`שחיי|גלישה|וסט פריחה` ואת `KW.dresses` ל-`חצאית`. מסיר ~380 התראות שווא — דוחות עתידיים נקראים.

**מאמץ/סיכון.** אפס סיכון, כלי פיתוח בלבד.

## מגבלות מקובלות (אין תיקון — אף פעם לא מציגות ערך שגוי)

- **Fox 0% צבע · Shilav 63% ללא צבע** — נופלים ל-`אחר`, אף פעם לא צבע שגוי. אין אות צבע במקור.
- **Shilav 95% דו-מגדרי** — בגדי תינוקות בסיסיים הם אכן יוניסקס; חוץ מבאג 2 (חצאיות) זה תקין.
- **NEXT 8% דו-מגדרי** — זה כפילות גיל (אותו מוצר ב-0-2 וגם 2-8), לא מגדר.

## המלצה שלי (סדר עדיפות לפיילוט)

1. **באג 1 (צבע)** — ההשפעה הכי גדולה על חוויית הסינון, וזה מה שדיווחת. לעשות ראשון.
2. **באג 2 (מגדר Shilav)** — סיכון נמוך, מונע את הבאג הכי מביך. לעשות יחד עם 1.
3. **ניקוי כלי האודיט** — זול, נותן לנו דוח אמין לאימות אחרי 1+2.
4. **באג 3 (קטגוריות TX/NEXT)** — קטן, אפשר לדחות לסבב הבא.

אחרי 1-3: להריץ מחדש את האודיט + לשחזר את מסנן החולצה-האדומה כהוכחה שנקי.
